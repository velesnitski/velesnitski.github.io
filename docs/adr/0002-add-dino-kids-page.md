# 0002 — Add a playful `/dino` page for the kids

**Date:** 2026-07-01
**Status:** Accepted

## Context

Wanted a small, fun sub-page off the landing site for my kids — a cute dinosaur they can play with. GitHub Pages maps folders to URL paths, so a new folder serves as a new page with no build step or router.

Requirements:

- Self-contained and dependency-free (no libraries, no network, works offline).
- Touch-first: must work on phones and tablets, not just desktop.
- Cheap to maintain and easy to restyle.

## Decision

Added `dino/index.html` — a single self-contained page served at `dino/` (i.e. `…/Landing/dino/`).

- The character is a chubby cartoon long-neck (sauropod) drawn entirely in code on a `<canvas>` — flat cel fills, a single silhouette outline (two-pass fill trick), cream underside, mint back-plates, curled tail.
- Interactive: tap / click / **space** makes it jump (simple velocity + gravity arc). Toggles for walk, 😎 shades, and a 👒 panama hat; live recolour of body / plates / belly.
- Shareable presets via URL hash (e.g. `#walk,shades,hat`).
- All asset links are relative (`../favicons/…`, `../`) so the page works whether Pages serves the repo at root or under `/Landing/`.

No hero art was generated by an image model — the sprite is authored as vector/canvas code, so every frame is consistent, transparent, tiny, and editable in plain English.

## Consequences

- New public page at `…/Landing/dino/`; linked back to home, not yet linked *from* home (kept as a low-profile page).
- Because it is pure canvas code, restyles ("longer neck", "different colour", new accessory) are one-line tweaks and cost nothing to ship.
- If it grows into a full endless-runner game, keep it in this same folder and split JS out only if the single file gets unwieldy. Supersede this record if the page moves to a dedicated `velesnitski.github.io` root repo.
